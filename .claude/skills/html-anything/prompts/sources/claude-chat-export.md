# Claude chat export

This is a Claude / Anthropic-style chat export. The shared
`_ai_chat_export.md` family prompt covers the page contract — read
that first; this file only adds Claude-specific framing.

## Export instructions

If the user is asking for help getting their data:

1. Sign in at claude.ai.
2. Settings → Account → **Export data** (or "Export account data").
3. Anthropic emails a `.zip` within minutes. Inside there's a
   `conversations.json` (the chat history) and `users.json`
   (the account profile).
4. Drop `conversations.json` into html-anything.

Some users may also have organization / project exports — those
follow the same `chat_messages` shape and route through the same
parser.

## Claude-specific framing

- The platform label is "Claude". Model values may include
  `claude-opus-4`, `claude-sonnet-4-6`, `claude-haiku-4-5`,
  `claude-3-5-sonnet`, etc.
- Each conversation in the Claude export carries `chat_messages`
  with `sender: "human" | "assistant"` instead of the OpenAI-style
  `role: "user" | "assistant"`. The parser normalizes both to
  `role` — don't try to special-case it in the page.
- Claude's `name` field is the conversation title (often a
  user-typed name, or auto-generated by the assistant). Keep it
  verbatim — don't truncate or rephrase.
- `created_at` / `updated_at` are ISO strings, already converted
  to epochs by the parser.
- Claude conversations may include attachments (`attachments` field
  on a message). The parser does not surface attachment bodies —
  if `attachmentCount` data appears in `meta`, it's a counter only;
  don't allocate a major section to attachments unless the user's
  history is attachment-heavy.

Stick to the required six sections from `_ai_chat_export.md`.
