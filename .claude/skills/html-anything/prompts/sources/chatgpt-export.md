# ChatGPT export (`conversations.json`)

This is OpenAI's `conversations.json` from the ChatGPT data export.
The shared `_ai_chat_export.md` family prompt covers the page
contract — read that first; this file only adds ChatGPT-specific
framing.

## Export instructions

If the user is asking for help getting their data:

1. Sign in at chat.openai.com.
2. Settings → Data controls → **Export data** → "Confirm export".
3. ChatGPT emails a `.zip` within minutes. Inside, the file is
   `conversations.json` (often hundreds of MB for heavy users).
4. Drop that file into html-anything.

Mention that the export also contains `chat.html`, `message_feedback.json`,
`model_comparisons.json`, and `user.json` — html-anything currently
focuses on `conversations.json` (the actual chat history).

## ChatGPT-specific framing

- The platform label is "ChatGPT". The model breakdown will usually
  show `gpt-4o`, `gpt-4-turbo`, `o1-preview`, `gpt-4`, etc.
  Render those slugs verbatim — don't try to map them to friendly
  names; the slug is what the user sees in the ChatGPT settings.
- ChatGPT conversations have a `mapping` graph (parent / children),
  the parser already linearized it to the canonical leaf-to-root
  thread, so each conversation is a single timeline. Don't try to
  show "branches" — the user does not see them in the ChatGPT UI
  either.
- Conversation titles are auto-generated by ChatGPT after the
  first turn. Keep them in the index even when generic ("New chat")
  — `topic` already falls back to keyword stems for those.
- ChatGPT exports do NOT carry per-message tool / function-call
  trace details in a useful form — `tool` role messages will be
  rare or empty; don't allocate a major section to them.

Stick to the required six sections from `_ai_chat_export.md`.
