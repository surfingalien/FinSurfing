# Default Idea.md Template

Use this when the repo has no existing `Idea.md` convention.

```md
# <Idea Name>

## Overview

One short paragraph explaining the idea in plain English.

## Problem

What is broken, missing, slow, expensive, or frustrating today?

## Who It's For

Who experiences the problem most acutely?

## Proposed Solution

What should exist instead?

## Why This Could Work

What is different or promising about this approach?

## Repo / Technical Fit

If a repo exists:
- what existing systems, packages, or patterns this can reuse
- major constraints or dependencies

If there is no repo, replace this with key technical assumptions.

## MVP

What is the smallest version worth validating?

## Risks and Open Questions

- Unknowns
- tradeoffs
- adoption risks
- technical or market risks

## Validation Plan

What should be tested, researched, prototyped, or confirmed next?
```
